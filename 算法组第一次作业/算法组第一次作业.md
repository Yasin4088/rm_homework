**TRANSISTOR**

---
这份作业旨在通过一个简单的 C++ 项目，带你走通一个简单完整工作流程：
代码编写 -> 构建系统 -> 版本控制 -> 文档规范。

在 Transistor，这样的任务到处都是，因此挑战一下自己，尽量在不使用 ai **“一步到位"** 的情况下，完成本次作业。

提示：这是一个md文件。对于 .md 文件，有专门的编辑器会让你能很舒服的看到作者想让你看到的格式。市面上流行的支持 md 的软件挺多，我个人推荐 obsidian。当然 vscode 本身也能在拓展里下载 md 相关插件，苹果的记事本也可以（安卓我不太了解）。无论如何，选一个你喜欢的。

如果你没使用相关软件，你可能会看到 [[]]，其中的内容是该文件夹中包含的图片，自行查看即可。

## 0. 配置好你的 Ubuntu22.04 环境 
## 1. 目标
1.  Git：掌握 git 的简单操作，理解 github 的使用，远程与本地的关系。
2.  CMake：学习管理多文件夹结构的工程构建。
3.  Markdown：试着编写标准、专业的项目说明文档 (README.md)。让别人看懂你的代码，是作为程序员最基本也是最重要的技能。

## 2. 第一阶段：Git

1.  克隆模板
```
git clong 对应的链接
```
2. 将作业仓库 fork 到你自己的仓库下![[Fork.png]]
3. 修改远端为自己的仓库
```
	git remote set-url 你fork的仓库的链接
	git remote -v 
```
第二条指令会输出你现在指向的远端仓库，确保它是你的新仓库的链接。

## 3. 第二阶段：工程目录组织
你需要手动创建以下目录结构（试着用用终端指令），严禁将所有文件堆放在根目录：

```text
.
├── include/         # 存放头文件 (.hpp)
├── src/             # 存放源代码 (.cpp)
├── CMakeLists.txt   # CMake 配置文件
├── .gitignore       # Git 忽略规则
└── README.md        # 项目文档
```
BTW：终端中输入下列指令可以让你看到类似上图的图像

```bash
sudo apt install tree
tree .
```

### 代码要求：
*   `include/factorial.h`：声明阶乘计算函数 `long long calculate(int n);`。
*   `src/factorial.cpp`：实现上述函数逻辑，并 include 上述文件。
*   `src/main.cpp`：调用函数并在终端打印输出用户输入的数的阶乘（使用 cin 与 cout）。
```c++
std::cin << 
```
*  本作业并不着重考察算法能力，所以能用就行。
* 注意代码规范，丢掉坏习惯：不要使用 x, y, z 命名，最好是不需要注释，代码就能解释自己。这需要你在命名上有些讲究。
---

## 4. 第三阶段：构建与自动化配置

### 1. CMakeLists.txt
要求使用 CMake 管理工程，核心配置必须包含：
```cmake
cmake_minimum_required(VERSION 3.8)
project()#你起的名字
```
 不要手写 build 文件夹：让 CMake 自动生成它，你的代码库应该保持干净。

### 2. .gitignore（环境清理）
确保 `build/` 文件夹内的中间文件不会被提交，编写 .gitignore 来实现这一点。

## 5. 第四阶段：编写 README.md

一份及格的文档应包含：
1.  项目简介：简单描述这是什么。
2.  项目树展示：使用 Markdown 代码块展示你的目录结构。
3.  如何运行：写明环境要求及运行命令（`bash run.sh`）。
4.  \*可以在readme最后提问，也可以直接在群里问或者私信。欢迎问题，但在那之前，可以耐心看看 [提问的智慧]([How-To-Ask-Questions-The-Smart-Way/README-zh_CN.md at main · ryanhanwu/How-To-Ask-Questions-The-Smart-Way](https://github.com/ryanhanwu/How-To-Ask-Questions-The-Smart-Way/blob/main/README-zh_CN.md))。

## 6. 提交方式
1.  在本地完成开发并测试通过
2.  执行 Git 提交到远程仓库
	   算法组组长强烈推荐小网站 [gitmoji | git 提交信息的 emoji 指南](https://gitmoji.js.org/)
3. 在 github 上执行 Contribute $\to$ Pull Request（简称PR）
![[PR.png]]

提示：git commit 的提示信息也要让别人一眼看懂你干了什么。我会推荐小步快跑，即修改一小处，就提一次 commit 存档。存档多，回档的点才多，不要害怕存档。
